已清仓股票统计功能 - 后端开发需求
技术栈：Python FastAPI + MySQL
一、项目背景与目标
本功能基于原始交割单表（original_delivery），对所有历史上已完成完整买卖周期（即累计持仓归零）的股票进行统计分析，按周期维度展示盈亏情况。核心目标：将原始的买卖流水记录，通过周期切割算法，还原为一个个完整的建仓 - 清仓周期，并对每个周期计算盈亏、收益率、持仓天数等指标。

三、核心算法：周期切割
3.1 算法说明
对每只股票，按 trade_date ASC、trade_time ASC（NULL 排最后）的顺序扫描所有交割记录，维护一个累计持仓数量 holding_qty。每当 holding_qty 归零时，认为一个完整的买卖周期结束，将当前周期的记录切割存入结果列表，然后开始新的周期。
3.2 Python 实现参考
python
运行
def split_into_cycles(records: list[dict]) -> list[list[dict]]:
    """
    records: 已按 trade_date, trade_time 排序的某只股票的所有记录
    返回: 切割好的周期列表，每个元素是该周期内所有记录的列表
    """
    cycles = []
    current_cycle = []
    holding_qty = 0

    for record in records:
        current_cycle.append(record)
        if record['trade_type'] == '买入':
            holding_qty += record['quantity']
        else:  # 卖出
            holding_qty -= record['quantity']

        if holding_qty == 0:  # 周期结束
            cycles.append(current_cycle)
            current_cycle = []

    # holding_qty != 0 的尾部记录属于当前持仓，忽略（已清仓统计不含）
    return cycles
3.3 周期统计指标计算
对每个切割出来的周期 cycle（一个 records 列表），按以下公式计算指标：
表格
指标字段名	计算公式	说明
stock_code	records[0]['stock_code']	股票代码
stock_name	records[0]['stock_name']	股票名称
cycle_index	该股票第 N 个周期（从 1 开始）	周期序号
open_date	min (r ['trade_date'] for r if r ['trade_type']==' 买入 ')	建仓日期（首次买入日）
close_date	max (r ['trade_date'] for r if r ['trade_type']==' 卖出 ')	清仓日期（最后卖出日）
hold_days	(close_date - open_date).days	持仓天数
buy_count	len ([r for r if r ['trade_type']==' 买入 '])	买入次数
sell_count	len ([r for r if r ['trade_type']==' 卖出 '])	卖出次数
total_buy_amount	sum (r ['deal_amount'] for r if 买入)	累计买入成交金额
total_sell_amount	sum (r ['deal_amount'] for r if 卖出)	累计卖出成交金额
total_buy_qty	sum (r ['quantity'] for r if 买入)	累计买入数量
profit_loss	sum(r['occur_amount'] for r in cycle)	净盈亏（含手续费）
profit_rate	profit_loss / total_buy_amount * 100	收益率（%），保留 2 位小数
avg_buy_price	total_buy_amount / total_buy_qty	买入均价
3.4 成本百分比计算（关联行情表）
关联 stock_daily_quote 表，使用建仓日期（open_date）查询当天开盘价和收盘价，关联查询语句：
sql
SELECT open_price, close_price
FROM stock_daily_quote
WHERE stock_code = :stock_code AND trade_date = :open_date
（可在业务层组装或直接 JOIN）
相关指标计算：
表格
指标字段名	计算公式	说明
cost_vs_open	(avg_buy_price - open_price) / open_price * 100	买入均价相对建仓日开盘价的偏差（%）
cost_vs_close	(avg_buy_price - close_price) / close_price * 100	买入均价相对建仓日收盘价的偏差（%）
💡 正数表示买入均价高于当天开 / 收盘价（买在偏高位），负数表示买入均价低于当天开 / 收盘价（买在偏低位）。💡 若 stock_daily_quote 中无对应日期数据，则 cost_vs_open 和 cost_vs_close 返回 null，前端展示为「-」。

五、数据处理流程
5.1 已清仓列表查询流程
表格
步骤	操作	说明
1	查询全部交割记录	SELECT * FROM original_delivery ORDER BY stock_code, trade_date ASC, trade_time ASC NULLS LAST
2	按 stock_code 分组	在 Python 中用 itertools.groupby 或字典分组
3	对每只股票调用 split_into_cycles ()	得到该股票的所有周期列表
4	过滤当前仍持仓的尾部记录	split_into_cycles 返回的是已归零周期，自动排除未清仓部分
5	计算每个周期的统计指标	按第三节公式计算各项指标
6	JOIN 行情表补充成本百分比	用 open_date 关联 stock_daily_quote
7	应用筛选、排序、分页	在 Python 中对结果列表做 filter /sort/slice
8	返回 JSON 响应	
💡 由于数据量仅数万条，步骤 1 一次性加载全部数据到内存完全可行，无需复杂的分批处理。

七、开发注意事项
持仓数量使用整数（int）比较，避免浮点误差导致归零判断失败。
所有金额字段保留 2 位小数，收益率保留 2 位小数。
排序时 NULL 值排在最后（MySQL 默认行为，ASC 排序 NULL 最小，需用 ISNULL (trade_time) ASC 处理）。
若建仓日期在 stock_daily_quote 中无对应数据，cost_vs_open /cost_vs_close 字段返回 null，前端显示「-」。
cycle_index 不存储在数据库中，每次查询时按 open_date 排序动态生成，从 1 开始计数。
行情同步接口重复调用不应产生重复数据（依赖 UNIQUE KEY 保证）。
