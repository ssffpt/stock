# 前端页面设计 - 清仓数据管理

## 1. 页面概述

### 页面名称
清仓数据管理

### 路由
`/cleared-positions`

### 页面功能
管理已清仓股票的预计算数据，支持手动触发增量清洗、查看清洗状态和历史日志。

### 位置
在现有「已清仓股票统计」页面基础上改造，入口不变。

## 3. 组件详细设计

### 3.1 清洗控制区 (CleanControl)

#### 布局
横向排列，左侧为时间范围和股票选择，右侧为清洗按钮。

#### 时间范围选择器
- 三个快捷按钮：本月、近三月、自定义
- 自定义模式下显示两个日期输入框
- 默认选中：本月

#### 股票选择器
- 输入框，支持多股票代码输入
- 占位符提示：「留空表示清洗所有股票」
- 支持逗号分隔输入

#### 清洗按钮
- 文本：「开始清洗」
- 状态：
  - 默认：蓝色背景，白色文字
  - Loading：显示旋转图标 + 「清洗中...」
  - 禁用：灰色背景，不可点击

#### 清洗状态指示器
- 空闲时：隐藏
- 清洗中：显示进度文字
- 完成后：显示成功提示，3秒后消失

### 3.2 清洗状态显示区 (CleanStatus)

#### 布局
横向三个统计卡片

#### 显示内容
| 指标 | 说明 |
|------|------|
| 最后清洗时间 | 格式：2026-03-27 10:30:00，未清洗过显示「从未清洗」 |
| 缓存股票数 | 已预计算的股票数量 |
| 缓存周期数 | 已预计算的周期总数 |

### 3.3 清仓列表区 (ClearedPositionTable)

#### 筛选栏
| 控件 | 类型 | 默认值 |
|------|------|--------|
| 股票代码 | 输入框 | 空 |
| 盈亏筛选 | 下拉选择 | 全部 |
| 日期快捷选 | 按钮组 | 本月 |

#### 数据表格
沿用现有表格设计，列定义不变：

| 列名 | 字段 | 对齐 |
|------|------|------|
| 股票代码 | stock_code | 左 |
| 股票名称 | stock_name | 左 |
| 建仓日期 | open_date | 左 |
| 清仓日期 | close_date | 左 |
| 持仓天数 | hold_days | 右 |
| 买入均价 | avg_buy_price | 右 |
| 买入金额 | total_buy_amount | 右 |
| 卖出金额 | total_sell_amount | 右 |
| 盈亏 | profit_loss | 右 |
| 收益率 | profit_rate | 右 |
| 操作 | 操作按钮 | 居中 |

#### 分页组件
沿用现有设计。

### 3.4 清仓明细弹窗 (PositionDetailModal)

沿用现有弹窗设计，入口为「查看明细」按钮。

### 3.5 清洗历史弹窗 (CleanHistoryModal)

#### 触发方式
点击状态区的「查看历史」链接

#### 内容
| 列名 | 说明 |
|------|------|
| 清洗时间 | 操作时间 |
| 日期范围 | start_date ~ end_date |
| 股票数 | stock_count |
| 周期数 | cycle_count |
| 状态 | success/failed/running |

## 6. 交互流程

### 6.1 清洗流程

```
1. 用户选择日期范围和股票（可选）
2. 点击「开始清洗」
3. 按钮变为 Loading 状态
4. 显示进度提示（轮询或 WebSocket）
5. 清洗完成，显示成功提示
6. 刷新清洗状态显示区
7. 刷新清仓列表
```

### 6.2 查看历史

```
1. 用户点击「查看历史」
2. 弹出清洗历史弹窗
3. 显示最近清洗记录列表
4. 支持分页
```

## 7. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 清洗接口调用失败 | 显示错误提示，按钮恢复可点击 |
| 清洗进行中再次点击 | 按钮禁用，提示「清洗进行中」 |
| 查询状态失败 | 显示上次成功获取的状态，隐藏错误 |
| 列表查询失败 | 显示错误提示，可重试 |
