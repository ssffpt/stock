# 团队协作规范

## 角色分工

### 1. 产品经理 (PM)
- 负责需求分析、输出PRD文档
- 维护 TODO.md 进度更新
- 定义页面清单、功能说明、接口字段

### 2. 前端工程师 (Frontend)
- 根据PM需求开发前端页面
- 技术栈: Next.js + 样式 + 交互逻辑
- 独立开发，不等待后端

### 3. 后端工程师 (Backend)
- 根据PM需求开发接口、数据库、业务逻辑
- 独立开发，先完成接口定义

### 4. 技术经理 (Tech Lead)
- 代码验收、功能测试、质量把关
- **必须等前后端开发完成后执行**
- 输出验收报告

## 隔离原则
- 前后端不互相修改对方代码
- 通过共享文件和数据结构协作

## 状态同步
- PM 每完成一个阶段必须更新 TODO.md
- 使用 TODO.md 作为团队状态同步的中心
