# 股票复盘系统 - 项目文档

## 1. 项目概述

股票复盘系统是一个基于 FastAPI + Next.js 构建的股票交易复盘管理系统，用于管理股票交割单、分析已清仓股票周期的盈亏情况，并支持行情数据同步。

### 1.1 核心功能

- **原始交割单管理**：支持单条/批量导入股票交割记录，提供分页查询、筛选功能
- **已清仓股票统计**：自动计算每个买卖周期的盈亏指标，支持多维度筛选排序
- **行情数据同步**：从东方财富API同步股票日行情数据，计算成本偏差

## 4. 核心业务逻辑

### 4.1 周期切割算法

系统将每只股票的所有交割记录按时间顺序切割成完整的买卖周期：

```python
def split_into_cycles(records):
    cycles = []
    current_cycle = []
    holding_qty = 0

    for record in records:
        current_cycle.append(record)
        if record['trade_type'] == '买入':
            holding_qty += record['quantity']
        else:
            holding_qty -= record['quantity']

        if holding_qty == 0:
            cycles.append(current_cycle)
            current_cycle = []

    return cycles
```

**逻辑说明：**
1. 按股票代码、成交日期、成交时间排序所有交割记录
2. 买入时增加持仓数量，卖出时减少持仓数量
3. 当持仓数量归零时，标志一个完整周期结束
4. 未清仓的尾部记录不计入已清仓统计

### 4.2 统计指标计算

每个已清仓周期计算以下指标：

| 指标 | 说明 |
|------|------|
| open_date | 建仓日期 (首次买入日期) |
| close_date | 清仓日期 (最后卖出日期) |
| hold_days | 持仓天数 (清仓日期 - 建仓日期) |
| buy_count | 买入次数 |
| sell_count | 卖出次数 |
| total_buy_amount | 买入金额合计 |
| total_sell_amount | 卖出金额合计 |
| profit_loss | 总盈亏 (所有发生金额之和) |
| profit_rate | 收益率 (盈亏/买入金额 * 100) |
| avg_buy_price | 买入均价 (买入金额/买入数量) |
| cost_vs_open | 成本偏差(开盘) = (买入均价 - 开盘价) / 开盘价 * 100 |
| cost_vs_close | 成本偏差(收盘) = (买入均价 - 收盘价) / 收盘价 * 100 |

### 5.2 页面功能

#### 已清仓股票页面 (`/`)

- **筛选功能**：股票代码搜索、盈亏筛选 (全部/盈利/亏损)、日期范围 (本月/近三月/自定义)
- **排序功能**：支持按清仓日期、盈亏金额、收益率、持仓天数、建仓日期排序
- **数据表格**：展示股票代码、名称、周期、日期、交易次数、均价、金额、盈亏、收益率、成本偏差
- **明细弹窗**：点击查看某周期的所有买卖记录明细

#### 原始交割单页面 (`/original-delivery`)

- **筛选功能**：股票代码、股票名称、日期范围筛选
- **数据表格**：展示成交日期、时间、股票信息、买卖类型、数量、价格、金额、手续费、备注
- **新增功能**：弹窗表单手动添加交割记录
- **批量导入**：支持 Excel 文件批量导入，自动解析并预览

## 9. 数据流

```
用户上传Excel → 前端解析 → 批量导入API → 存入original_delivery表
                                              ↓
                              周期切割算法 → 计算统计指标 → 展示已清仓统计
                                              ↓
                              同步行情API → 东方财富API → 存入stock_daily_quote表
                                              ↓
                              关联行情 → 计算成本偏差 → 完整展示
```
